Relatorio Exercicio Pratico 04 ULA 4 bits + Arduino

1. Objetivo

Desenvolver um programa que leia instruções de um arquivo ( `teste.ula` ), 
processe os valores de entrada e gere um arquivo de saída ( `testeula.hex` ) compatível com o hardware Arduino.

-----------------------------

2. Lógica de Funcionamento

Para processar os dados, utilizamos duas pilhas:

1) Leitura ( Pilha 1 ): O arquivo é lido linha por linha, cada instrução é empilhada na Pilha 1.

2) Ordenação ( Pilha 2 ): Como a última linha lida fica no topo, passamos tudo da Pilha 1 para a Pilha 2. 
Isso inverte a ordem, deixando a primeira instrução do arquivo pronta para ser a primeira a sair.

3) Execução: O programa retira as instruções da Pilha 2 uma a uma para processar.


Exemplo:

    Pilha 1            Pilha 2

|            |     |            |              
|    fim.    |     |    inicio: |                   Se: w=AB
|    W=Bn    |     |    X=C     |                         W -> Hexa = B 
|    X=D     |     |    Y=6     |                         X - > C
|    W=nAeB  |     |    W=AB    |                         Y - > 6 
|    W=AoB   |     |    X=A     |                   Se: w=' '
|    Y=3     |     |    Y=3     |                         Erro na linha   
|    X=A     |     |    W=AoB   |
|    W=AB    |     |    W=nAeB  |
|    Y=6     |     |    X=D     |           
|    X=C     |     |    w=Bn    |
|   inicio:  |     |    fim.    |
|____________|     |____________|

-----------------------------

3. Processamento de Dados

O compilador realiza as seguintes tarefas em cada linha:

- Identificação: Reconhece o que é valor de entrada ( X e Y ) e o que é operação de saída ( W ).
- Cálculo: Realiza a operação lógica solicitada ( ex: CopiaA, AeBn , AeB ).
- Conversão: Transforma o resultado e os comandos em formato Hexadecimal.

-----------------------------

4. Resultado Final

O arquivo `testeula.hex` gerado contém a sequência de códigos que o Arduino deve seguir para acionar os LEDs ou circuitos da ULA de 4 bits, 
garantindo que o comportamento digital coincida com o que foi escrito no arquivo de teste.

-----------------------------

MYIO.H — O QUE CADA FUNÇÃO FAZ

imprimirPilha(Pilha* pilha)
    Percorre a pilha do topo ao fundo e imprime cada string
    no terminal. Usada para debug.

print / println
    Funções auxiliares para imprimir uma string com ou sem
    quebra de linha no final.

printErroLinha(const char* frase, int indiceLinha)
    Imprime uma mensagem de erro com o número da linha e o
    conteúdo que causou o problema.

trocarMne(char palavra[])
    Função central. Recebe uma linha no formato 'V=mnemonico;'
    e retorna o caractere hex correspondente ao mnemônico.
    Compara os caracteres a partir da posição 2 para identificar
    qual mnemônico é. Se não reconhecer nada, retorna espaço ' '.

linhaValida(const char* linha)
    Verifica se a linha termina com ';'.
    Retorna true se válida, false caso contrário.

criarPilha(void)
    Aloca e retorna uma nova pilha vazia.

push(Pilha* pilha, const char* str)
    Insere uma string no topo da pilha usando strdup para
    garantir uma cópia independente.

pop(Pilha* pilha)
    Remove e retorna a string do topo. O chamador deve dar
    free() na string retornada. Retorna NULL se vazia.

pilhaVazia(Pilha* pilha)
    Retorna true se o topo for NULL.

-----------------------------

COMPILADOR.C — O QUE CADA FUNÇÃO FAZ

lerLinhas(FILE* arquivo_fp)
    Lê o arquivo .ula e retorna as instruções na ordem
    correta (inicio -> fim).

    Passo a passo:
    1. Cria pilhaA e pilhaB
    2. Descarta a primeira linha 'inicio:' com fgets
    3. Lê linha por linha removendo o '\n' com strcspn
    4. Para quando encontra 'fim.'
    5. Se a linha for válida empilha em pilhaA
    6. Se inválida chama printErroLinha e para
    7. Transfere tudo de pilhaA para pilhaB (essa inversão
       faz pilhaB.topo apontar para a primeira instrução)
    8. Libera pilhaA e retorna pilhaB

    Por que duas pilhas? A pilhaA guarda as linhas invertidas
    (última lida no topo). Ao transferir para pilhaB inverte
    novamente, voltando à ordem original do arquivo. Assim
    gerarArquivoHex pode processar em ordem sem precisar de
    nenhuma estrutura auxiliar extra.

gerarArquivoHex(Pilha* pilhaResultado, FILE* arquivoHex)
    Processa a pilhaB já na ordem correta e grava no .hex.

    Passo a passo:
    1. Declara registradores X e Y iniciados com ' '
    2. Desempilha elemento a elemento
    3. Se começa com 'X': chama trocarMne e salva em X
    4. Se começa com 'Y': chama trocarMne e salva em Y
    5. Se começa com 'W': chama trocarMne e grava XYW no hex
    6. Libera cada elemento após processar

main(void)
    1. Abre test.ula para leitura
    2. Abre test.hex para escrita
    3. Chama lerLinhas para obter a pilhaB ordenada
    4. Fecha o .ula
    5. Chama gerarArquivoHex para processar e gravar
    6. Fecha o .hex e libera a pilha
